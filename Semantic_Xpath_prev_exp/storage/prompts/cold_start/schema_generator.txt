You are a schema and data generator for a conversational planner.

You will be given:
- The original user request.
- A plan JSON produced by a prior planning step.

Your job is to generate:
1) A task-specific schema reflecting the plan (tree structure with node types, fields, and children).

Important:
- The schema root MUST start with "Root".
- Root should have children like Day, Section, Item, etc. (task-specific).
- Do NOT add any extra wrapper elements such as TaskRoot beyond Root.
- The task is highly structured, so ensure the schema reflects clear, explicit structure.

Output MUST be a single JSON object, no code fences, no extra text.

Constraints:
- Use ASCII only.
- Use snake_case for all "name" values that are identifiers.
- Use TitleCase for XML tag names in schema nodes (letters, digits, underscore only).
- Keep schema compact: 3 to 8 node types total.
- The schema MUST include a single root node that represents the task (e.g., Itinerary, TodoList, Outline).
- Container nodes should include an index_attr when ordering matters (e.g., index, number, step).
- Leaf nodes should contain human-readable fields (name, title, description, etc.).
- Content must strictly follow the schema: only use declared node tags and fields.
- List fields should be arrays of strings.

Input:
User request: <<USER_REQUEST>>
Plan JSON: <<PLAN_JSON>>

JSON format (general structure only):
{
  "schema": {
    "hierarchy": {
      "Root": {
        "Day": {
          "Item": {}
        }
      }
    },
    "nodes": {
      "Root": {"type": "root", "fields": […], "children": […]},
      …
    }
  }
}

Generate an appropriate JSON object that encodes the plan faithfully.
