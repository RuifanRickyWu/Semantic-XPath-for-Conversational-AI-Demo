Output Format
====================
Output a single JSON object in the following format:
```json
{
  "updates": [
    {
      "id": "node_id",
      "path": "Itinerary > Day X > Node Name",
      "fields": { "...": "..." },
      "new_type": null,
      "reasoning": "Brief explanation of the update."
    }
  ],
  "reasoning": "Overall explanation of the update decision."
}
If no nodes should be updated, output:
{
  "updates": [],
  "reasoning": "Explanation of why none of the candidates clearly match the request."
}
====================
Example
====================
User request:Change CN Tower visit to 2pm
Candidate nodes:
[1] Day 1 > CN Tower (POI, time_block: 11:00 AM – 12:30 PM)
Output:
{
  "updates": [
    {
      "id": "1",
      "path": "Day 1 > CN Tower",
      "fields": {
        "name": "CN Tower",
        "time_block": "2:00 PM - 3:30 PM",
        "description": "Visit one of the world's tallest towers with panoramic city views.",
        "travel_method": "Walk",
        "expected_cost": "CAD 43",
        "highlights": ["Glass floor", "360 Restaurant", "EdgeWalk"]
      },
      "new_type": null,
      "reasoning": "Changed time from 11:00 AM to 2:00 PM while preserving the original duration."
    }
  ],
  "reasoning": "Updated the CN Tower visit time. All other fields were preserved."
}
