- Do NOT explain your reasoning.

Itinerary (root)
├── Day (container)
│   ├── POI (leaf)
│   └── Restaurant (leaf)

IMPORTANT STRUCTURAL RULES:
- POI and Restaurant are SIBLINGS (same level under Day).
- You CANNOT have a path like /Day/POI/Restaurant - this is INVALID.
- Each query can only target ONE leaf node type (either POI or Restaurant, not both).

Path rules:
- Paths are symbolic and deterministic.
- You may ONLY use the allowed path node types: Itinerary, Day, POI, Restaurant.
- You may use positional indexing on Day nodes, e.g., Day[1], Day[2].

- You may use -1 to mean the last element, e.g., POI[-1].

- You may use parentheses for global indexing, e.g., (/Itinerary/Day/POI)[5].
- Do NOT put any free-text or semantic content in the path.

ALL semantic predicates MUST use all() or exists() wrapper. NO direct predicates allowed.

NEVER USE DIRECT PREDICATES:
❌ POI[description =~ "museum"]           - INVALID
✓ POI[all(description =~ "museum")]       - VALID
✓ POI[exists(description =~ "museum")]    - VALID

CHOOSING BETWEEN exists() AND all():
- For LEAF nodes: both are equivalent, prefer all()
- For CONTAINER nodes:
  - exists() = "Does any child match?" → "days with a museum"

IMPORTANT: When user asks for "Nth [semantic type]", include BOTH predicate AND index.
