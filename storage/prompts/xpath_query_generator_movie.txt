## Schema Structure

MovieDatabase (root)

## Query Syntax

### Path Navigation
- Paths use XPath-style navigation: /MovieDatabase/Genre/Movie
- Only use allowed node types: MovieDatabase, Genre, Movie
- Genre contains movies as children

### Index Notation
- Positional index: Movie[2] - select 2nd Movie
- Negative index: Movie[-1] - select last Movie
- Range index: Movie[1:3] - select Movie 1 to 3 (inclusive)
- Global index: (/MovieDatabase/Genre/Movie)[5] - select 5th Movie globally

### Semantic Predicates

THREE types of semantic predicates:

1. **sem()** - Score the node's OWN content only
   - Use when the property can be determined from the node itself
   - Syntax: sem(content =~ "value")
   - Example: Movie[sem(content =~ "space exploration")] - movies about space

2. **exist()** - At least one child matches (Noisy-OR aggregation)
   - Use when checking if ANY child of a container has a property
   - Syntax: exist(ChildType[sem(content =~ "value")])
   - Example: Genre[exist(Movie[sem(content =~ "award winning")])] - genres with at least one award-winning movie

3. **mass()** - General prevalence among children (Beta-Bernoulli aggregation)
   - Use when the property characterizes the container as a whole
   - Syntax: mass(ChildType[sem(content =~ "value")])
   - Example: Genre[mass(Movie[sem(content =~ "dark")])] - dark/serious genres

### Logical Operators
- AND: sem(content =~ "X") AND sem(content =~ "Y")
- OR: sem(content =~ "X") OR sem(content =~ "Y")
- Aggregation AND/OR: exist(...) AND exist(...), exist(...) OR mass(...)

## Decision Guide: When to Use Each Predicate

| Query Pattern | Predicate | Reasoning |
|--------------|-----------|-----------|
| "sci-fi movie" | sem() | Property of the movie itself |
| "Christopher Nolan film" | sem() | Property of the movie itself |
| "genre with horror movies" | exist() | Checking if any child is horror |
| "dark genre" | mass() | Overall character of the genre's movies |
| "critically acclaimed genre" | mass() | General quality of movies in genre |
