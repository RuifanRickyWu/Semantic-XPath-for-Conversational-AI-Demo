IMPORTANT STRUCTURAL RULES:
- Concept and Exercise are SIBLINGS (same level under Course).

- Each query can only target ONE leaf node type (either Concept or Exercise, not both).

Path rules:
- Paths are symbolic and deterministic.
- You may ONLY use the allowed path node types: Curriculum, Course, Concept, Exercise.
- You may use positional indexing on Course nodes, e.g., Course[1], Course[2].
- You may use positional indexing on leaf nodes, e.g., Concept[2], Exercise[1].
- You may use -1 to mean the last element, e.g., Concept[-1].
- You may use RANGE indexing with colon syntax: [start:end] (inclusive on both ends).
- You may use "to end" RANGE indexing: [-N:] means "last N elements".
- You may use parentheses for global indexing.
- Do NOT put any free-text or semantic content in the path.

PREDICATE RULES (CRITICAL):

ALL semantic predicates MUST use all() or exists() wrapper. NO direct predicates allowed.

   - all() = "children generally match"

2. INSIDE all()/exists(), you can use AND/OR:

NEVER USE DIRECT PREDICATES:
❌ Concept[description =~ "neural network"]           - INVALID
✓ Concept[all(description =~ "neural network")]       - VALID
✓ Concept[exists(description =~ "neural network")]    - VALID

CHOOSING BETWEEN exists() AND all():
- For LEAF nodes: both are equivalent, prefer all()
- For CONTAINER nodes:
  - exists() = "Does any child match?" → "courses with machine learning content"
  - all() = "Are children generally like this?" → "advanced courses"
