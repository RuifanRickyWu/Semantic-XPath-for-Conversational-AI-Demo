## Examples

### Basic Queries
User: find all urgent tasks that need immediate attention
Output: /Project/Task[atom(content =~ "urgent")]

User: show me all tasks in the frontend project
Output: /Project[atom(content =~ "frontend")]/Task

User: what tasks are assigned to the backend team?
Output: /Project[atom(content =~ "backend")]/Task

### Aggregation Queries
User: which projects have at least one incomplete task?
Output: /Project[agg_exists(Task[atom(content =~ "incomplete")])]

User: find projects that contain any blocked tasks so I can help unblock them
Output: /Project[agg_exists(Task[atom(content =~ "blocked")])]

User: show me projects that have a bug or a critical issue that needs fixing
Output: /Project[agg_exists(Task[(atom(content =~ "bug") OR atom(content =~ "critical")) AND not(atom(content =~ "resolved"))])]

### Negation Queries
User: find tasks that are not blocked so I can work on them
Output: /Project/Task[not(atom(content =~ "blocked"))]

User: show me tasks that are either high priority or urgent but not yet started
Output: /Project/Task[(atom(content =~ "high priority") OR atom(content =~ "urgent")) AND not(atom(content =~ "in progress"))]

### Local Index Queries (per-parent selection)
User: show me all projects in version 2
Output: /Project

User: review everything in the first project
Output: /Project[1]/*

User: give me tasks from project 2 through project 4 so I can review them
Output: /Project[2:4]/Task

User: show me the last two tasks in each project
Output: /Project/Task[-2:]

### Global Index Queries (overall result selection)
User: show me the first 5 tasks across all projects
Output: (/Project/Task)[1:5]

User: what is the 3rd task overall in the todolist?
Output: (/Project/Task)[3]

User: find the last task in the entire todolist
Output: (/Project/Task)[-1]

User: show me the last 3 urgent tasks across all projects
Output: (/Project/Task[atom(content =~ "urgent")])[-3:]

### Descendant Axis Queries (schema: Project → Task → SubTask)
User: show me all subtasks in the latest version
Output: /Project//SubTask

User: find all tasks across all projects in the current version
Output: /Project//Task

User: which projects have urgent subtasks that need attention?
Output: /Project[agg_exists(//SubTask[atom(content =~ "urgent")])]

User: find tasks that have blocking subtasks preventing progress
Output: /Project/Task[agg_exists(SubTask[atom(content =~ "blocking")])]

User: show me all bug-related subtasks across the entire todolist
Output: /Project//SubTask[atom(content =~ "bug")]

### Wildcard Queries (select all children)
User: show me everything in the first project
Output: /Project[1]/*

User: review all items in the last project
Output: /Project[-1]/*

User: give me the first 5 items across all projects
Output: (/Project/*)[1:5]

### Combined Index and Predicate
User: find the first urgent task in each project
Output: /Project/Task[atom(content =~ "urgent")][1]

User: show me the second bug-related task overall
Output: (/Project/Task[atom(content =~ "bug")])[2]

### CRUD Operations
Operation: CREATE
User: add a new task for implementing the login page in the frontend project
Output: /Project[atom(content =~ "frontend")]
Reason: The query retrieves the frontend Project node as the insertion scope. The terms "login page" describe properties of the task to be created and therefore do not participate in retrieval.

Operation: CREATE
User: add a subtask for writing unit tests under the authentication task
Output: /Project/Task[atom(content =~ "authentication")]
Reason: The query retrieves the authentication Task node as the insertion scope. The phrase "writing unit tests" describes the subtask to be created and does not affect retrieval.

Operation: UPDATE
User: mark the login task as completed
Output: /Project/Task[atom(content =~ "login")]
Reason: The query retrieves the existing login Task as the target node to be modified. The new status (completed) is applied during the update step and does not affect retrieval.

Operation: DELETE
User: delete all completed tasks
Output: /Project/Task[atom(content =~ "completed")]

Operation: DELETE
User: remove the first task in the backend project
Output: /Project[atom(content =~ "backend")]/Task[1]
